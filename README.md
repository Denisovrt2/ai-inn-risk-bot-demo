# AI INN Risk Bot Demo

## Что это

AI INN Risk Bot — система анализа контрагентов по ИНН для задач экономической безопасности, первичного due diligence и поддержки управленческих решений.

Проект решает практическую задачу: не просто показать сведения о компании, а помочь понять, насколько безопасно с ней работать.

Система собирает и интерпретирует сигналы из государственных и корпоративных источников, формирует собственные признаки риска и выдает структурированный AI-отчёт для предварительной оценки компании.

## Почему этот проект сильный

Это не витрина поверх LLM и не “чат по данным”.

Система спроектирована как прикладной risk engine для задач:
- экономической безопасности
- первичного due diligence
- анализа контрагентов
- поддержки управленческих решений

Сильная сторона проекта — не просто сбор данных, а их интерпретация через собственную feature-логику:

- используются не сырые поля, а признаки риска
- судебные данные рассматриваются не только по количеству дел, а по роли компании, характеру споров и совокупному паттерну
- корпоративные и регистрационные сигналы оцениваются в логике устойчивости контрагента
- AI используется как слой интерпретации поверх risk features, а не как замена аналитической модели

Итоговая цель продукта — не показать информацию, а выдать осмысленный decision signal:
насколько безопасно работать с компанией и где именно находится зона риска.

## Какие данные используются

В рабочей версии проекта используются сигналы из нескольких типов источников, в том числе:
- государственные реестры
- данные ФНС
- судебная активность
- корпоративные изменения
- операционные и поведенческие сигналы
- финансовые и регистрационные факторы

Важен не только сам факт наличия данных, а их интерпретация в контексте риска.

## Задача системы

В реальном бизнесе проверка контрагента почти всегда упирается в три проблемы:

1. данные разрознены
2. данные слабо интерпретированы
3. нет единого сигнала для решения

Этот проект закрывает именно этот разрыв:
- собирает сигналы
- очищает и нормализует данные
- формирует собственные признаки риска
- строит скоринговую логику
- добавляет AI-слой для интерпретации результата

## Pipeline

1. Сбор данных
2. Очистка и нормализация
3. Формирование risk features
4. Скоринг риска
5. Генерация AI-отчёта

## Ключевая логика

Главная идея проекта — не генерировать текст с помощью AI.

Главная идея проекта:
- извлечь реальные сигналы риска
- формализовать их в собственные признаки
- сделать вывод объяснимым
- дать основу для управленческого решения

Именно поэтому AI здесь не подменяет систему, а работает как верхний слой интерпретации над структурированными данными и risk features.

## Что в этом проекте важно

- анализ контрагентов как прикладная задача
- фокус на экономической безопасности
- работа с государственными и судебными данными
- собственная feature-логика, а не только raw data
- explainable AI reporting
- backend-oriented архитектура

## Ключевой инженерный артефакт

В репозитории добавлен публичный demo-фрагмент инженерной логики:

`src/demo_logic/advanced_features_demo.py`

Этот файл показывает важную часть архитектуры системы:
- как базовые признаки не теряются между слоями обработки
- как risk signals поднимаются в advanced layer
- как формируется explainable input для скоринга и AI-отчёта

Смысл этого слоя не в “обогащении JSON”, а в том, чтобы:
- не потерять сильные сигналы риска
- устранить конфликты между слоями признаков
- сделать итоговую decision-support логику устойчивой и интерпретируемой

Это упрощённый публичный фрагмент production-подхода:  
data → features → advanced reconciliation → scoring → AI interpretation.
